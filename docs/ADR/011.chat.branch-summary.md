## Context

分支探索完成后，如果保留所有消息，主线的上下文会被支线细节淹没。需要一种机制，把分支"折叠"成紧凑的摘要，保留关键结论，丢弃探索过程。

候选方案：
- 用户手动折叠（显式控制）
- 超过 N 轮自动压缩（规则触发）
- focus 切走时自动压缩（事件触发）

## Decision

**默认手动折叠**，后续可加自动触发。

摘要格式：

```json
{
  "type": "summary",
  "branch_topic": "树 vs 图",
  "conclusion": "树更简单，D1 一个 parent_id 字段够用",
  "evidence_count": 3,
  "confidence": "high"
}
```

摘要由 LLM 生成（用分支内的完整消息作为输入），不是简单截断。

保留能力：
- 摘要节点可展开，恢复分支的完整消息
- 展开后 focus 可以重新进入该分支继续探索

## Consequences

优点：

- 主线上下文保持精简
- 摘要保留结论和证据数量，LLM 可以判断是否需要回溯展开
- 手动折叠给用户完全控制权

缺点：

- 摘要质量依赖 LLM（可能丢失关键细节）
- 手动折叠需要用户有"该折叠了"的意识（UI 可以提示）
