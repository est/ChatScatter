## Context

AI 聊天系统需要配置 API Key（OpenAI、Anthropic 等）。方案：

- 写死在 wrangler.toml 的 `[vars]` 里
- 使用 `.dev.vars` 文件（wrangler dev 自动读取）
- 每次部署时通过 `wrangler secret put` 设置
- 代码里硬编码

写死在 wrangler.toml 会提交到 git，不安全。每次部署改 key 太蠢。

## Decision

本地开发用 `.dev.vars` 文件（已加入 .gitignore），生产环境用 `wrangler secret put`。提供 `.dev.vars.example` 作为模板。

## Consequences

优点：

- API Key 不进 git 仓库
- 本地改 key 不需要部署，`wrangler dev` 自动读取
- 生产环境 secret 由 CF 平台加密存储

缺点：

- 团队成员需要各自维护自己的 `.dev.vars`
- 需要记得 `.dev.vars` 在 `.gitignore` 里
