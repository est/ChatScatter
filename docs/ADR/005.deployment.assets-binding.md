## Context

前端静态文件（HTML/CSS/JS）需要部署到 Workers 上。方案：

- wrangler.toml 配置 `[assets]` 目录
- Worker 代码里手写静态文件 serving 逻辑（fetch + MIME 映射）
- 使用 Cloudflare Pages 单独部署前端

## Decision

使用 wrangler.toml 的 `[assets]` 配置，指定 `directory = "./public"`。

## Consequences

优点：

- 平台自动处理静态文件的缓存和 MIME 类型
- Worker 代码只负责 API 路由，职责清晰
- 不需要手写文件 serving 代码
- `wrangler dev` 时自动 serve public/ 下的文件

缺点：

- 静态文件和 Worker 代码部署在同一个 bundle 里
- 如果后续前端需要 SSR，需要额外方案
