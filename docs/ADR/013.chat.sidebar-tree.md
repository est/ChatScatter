## Context

树结构的数据需要对应的 UI 来导航。侧边栏是天然的树形容器。

侧边栏需要表达：
- 当前在树的哪个位置（focus 位置）
- 有哪些分支（可展开/折叠）
- 分支的状态（已摘要 / 活跃中 / 未读）
- 主线和支线的视觉层次

## Decision

侧边栏 = 会话树的导航地图。

```
▾ 当前会话
  ├─ ✓ 想做一个聊天系统
  ├─ ✓ 先研究数据结构
  │   ├─ ◉ 树 vs 图？          ← 已摘要，◉ 标记
  │   └─ ◉ KV cache 影响？
  ├─ ▸ 好，开始设计 API         ← focus 在这
  └─ ...
```

视觉约定：
- `▸` 当前 focus 位置
- `✓` 已完成的消息
- `◉` 已摘要的分支（可点击展开）
- `○` 未读/新分支
- 缩进 = 树层级

交互：
- 点击任意节点 → 移动 focus → 上下文重建 → 右侧聊天区更新
- 右键节点 → 分支操作（重命名、折叠、删除）
- 分支节点双击 → 展开/折叠子树

## Consequences

优点：

- 用户随时知道"我在哪、我从哪来、我还去过哪"
- 分支状态一目了然
- 和数据结构 1:1 映射，无认知负担

缺点：

- 树深度超过 3-4 层时，缩进会吃掉很多横向空间（可用面包屑缓解）
- 移动端需要适配（折叠侧边栏 + 底部导航）
