## Context

前端需要实现：左侧树状菜单 + 右侧聊天框。候选方案：

- React / Vue / Svelte 等 SPA 框架
- 原生 HTML + CSS + Vanilla JS
- htmx 或 Alpine.js 等轻量方案

约束：

- 用户明确说"不采用 SPA"
- 项目初期，不想引入构建工具链（bundler、dev server 等）
- Cloudflare Workers 的 [assets] 可以直接托管静态文件

## Decision

使用原生 HTML + CSS + Vanilla JS。不做 SPA，每个页面独立加载，JS 按模块组织。

## Consequences

优点：

- 零构建步骤，public/ 目录直接部署
- 开发迭代快，改了刷新就生效
- 没有框架 lock-in

缺点：

- 没有组件化系统，复杂 UI 需要自己管理 DOM
- 状态管理需要手写
- 路由需要自己处理（如果后续需要多页面）
