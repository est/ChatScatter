## Context

Cloudflare Workers 原生 API 是 `fetch(request) => Response`，太原始。需要一个轻量路由框架。候选：

- Hono
- itty-router
- 原生 Workers API 手写路由
- Express（通过 compat 层）

要求：

- 体积小，Worker 包体不能大
- 原生支持 CF Workers 环境
- 路由 + 中间件够用就行
- TypeScript 支持好

## Decision

选择 Hono。

## Consequences

优点：

- 专为 edge runtime 设计，零 Node.js 依赖
- 内置 cors、错误处理等常用中间件
- TypeScript 类型推导好
- 包体小（~14KB）

缺点：

- 多了一个依赖（虽然很轻）
- 团队不熟的话需要学一下 API
